Logging¶

Updated on 13 June 2015 @author: Sunil Ghai

logger.views.log_a_success(user_id, request, response_code, ts, source_key=None, comments={})[source]¶

Logs a successful action on the website. Writes into ActionLogByUser and ActionLogBySource (if there is a source_key mentioned). We do not log response text here.

Parameters:
  • user_id – User id of the user
  • request – Request object
  • response_code – Response code
  • ts – Timestamp of the request arrival
  • source_key – Source key of the source (optional)
  • comments – Any additional comments as a dictionary (<text>:<text>) (optional)
Returns:

None
logger.views.log_an_error(user_id, request, response_payload, response_code, error_text, ts, success, source_key=None, comments={})[source]¶

Logs an error on the website for a known User (correspond to actions where User has to be logged in). Writes into ActionLogByError, ActionLogByUser and ActionLogBySource (if source_key is mentioned).

Parameters:
  • user_id – User id of the user
  • request – Request object
  • response_payload – Response (payload) as a text
  • response_code – Response code
  • error_text – Error text
  • ts – Timestamp of the request arrival
  • success – If the called URL was a success or not (mostly False)
  • source_key – Source key (optional)
  • comments – Any additional comments as a dictionary (<text>:<text>) (optional)
Returns:

None

logger.views.log_a_data_write_failure(user_id, request, response_payload, response_code, error_text, ts, source_key, comments={})[source]¶

Logs a data write failure. Writes into InvalidDataStorageBySource, DataWriteHistoryByUser, DataWriteHistoryBySource, ActionLogByError, ActionLogByUser and ActionLogBySource tables.

Parameters:
  • user_id – User id of the user
  • request – Request object
  • response_payload – Response (payload) as a text
  • response_code – Response code
  • error_text – Error object (from settings.ERRORS)
  • ts – Timestamp of the request arrival
  • source_key – Source key of the source (must be present)
  • comments – Any additional comments as a dictionary (<text>:<text>) (optional)
Returns:

None
