Parameters:
  • user_id – User id of the user
  • request – Http request received. Either from Django/Django REST framework.
  • ts – Timestamp at which the request was received
  • error – Error instance (settings.ERRORS)
  • response_type – Either to create a DRF response (Response) or a Django Response (HttpResponse)
  • use_template – If to use error template while rendering a response. Or plain JSON (further categorization
  • comments – Comments to be stored with the log
  • response_params – A list (key:value) of additional parameters to be defined in the response
  • source_key – source_key if the error is associated to a source.
Returns:

Returns an Http Response of type (as defined in the call).

utils.errors.generate_error_response(request, response_type, error, error_code, use_template, response_params)[source]¶

Generates an error response of HttpResponse (Django) type or of Response (Django Rest Framework) type. It either renders an error template (for errors encountered in GUI) or creates a response with a dictionary. The dictionary at least contains an ‘error’ key with error description. Additional (key:value) pairs can be passed to be included in the response.

Parameters:
  • request – Http request received. Either from Django/Django REST framework.
  • response_type – Either to create a DRF response (Response) or a Django JsonResponse. (settings.RESPONSE_TYPES)
  • error – Error object (from settings.ERROR)
  • error_code – status_code to be set in the response.
  • use_template – (True/False) If the error template (dataglen/message.html) should be used or plain response.
  • response_params – A list (key:value) of additional parameters to be defined in the response
Returns:

Returns a response
